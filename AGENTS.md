# AGENTS

Ты работаешь только в проекте `bali-2026-trip`, отдельном публичном сайте семейного маршрута на Бали.

## Hard Rules

- Не трогать бизнес-домен пользователя.
- Не трогать коммерческие репозитории пользователя.
- Не публиковать личный маршрут в чужих или бизнес-проектах.
- Не коммитить приватные документы: PDF-билеты, PNR, паспорта, номера билетов, точные приватные адреса, телефоны, скриншоты бронирований.

## Data Rule

`src/data/trip.json` - master source of truth.

Любое изменение маршрута считается неполным, если не проверены:

- карта;
- цифровые пины;
- порядок точек;
- даты баз;
- дневной план;
- переезды;
- описания мест.

Карта и текст не редактируются независимо. Если меняется маршрут в тексте, меняется и карта через тот же `trip.json`.

## Naming Rule

Нельзя оставлять непонятные географические или английские названия без короткого русского описания. Для каждого места нужно объяснить:

- что это;
- зачем туда ехать;
- как это ложится в день;
- есть ли ограничения для семьи и ребёнка.

## Workflow

1. Проверить `git status`.
2. Читать `README.md`, `PROJECT_SPEC.md`, `TASKS.md`, `UX.md`.
3. Перед изменением данных маршрута читать `src/data/trip.json`.
4. После изменения запускать `npm run build`.
5. Перед commit проверять diff, `.gitignore`, отсутствие секретов и приватных артефактов.
